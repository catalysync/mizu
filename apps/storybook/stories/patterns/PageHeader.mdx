# PageHeader

Top-of-page preamble. Composes breadcrumbs + title + description + actions + an optional toolbar row for filters/search.

## When to use

- Standalone pages that aren't a resource-detail page: "All customers", "Reports", "Settings", "Search"
- The one-per-page header every feature lives under

## When NOT to use

- Resource-detail pages — use `ResourceHeader` (it adds a status pill + meta row)
- Section titles inside a page — use `SectionHeader`
- App-level global navigation — use `AppHeader`
