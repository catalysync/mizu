# Button

The Button component is mizu's canonical interactive control. It wraps a native
`<button>` element with variant + size tokens, loading + disabled state, and an
intentional `Inaccessible` / `Accessible` pair so you can see the a11y trap for
icon-only buttons directly in Storybook.

## Usage

<Canvas of={ButtonStories.Primary} />

```tsx
import { Button } from '@aspect/react';

<Button variant="primary" size="md" onClick={() => save()}>
  Save changes
</Button>;
```

## Sizes

Three sizes plus a dedicated `icon` size for icon-only controls. The `icon`
size enforces square dimensions and expects an `aria-label`.

<Canvas of={ButtonStories.AllSizes} />

## Loading state

Loading is modeled as a first-class prop, not a caller responsibility. Setting
`loading` applies `data-loading`, `aria-busy`, and `disabled` together, and
absolute-positions a spinner over a hidden label so the button retains its
original width.

<Canvas of={ButtonStories.Loading} />

## Accessibility

- Icon-only buttons **must** have an `aria-label`. The `Inaccessible` story
  below is deliberately broken so the a11y addon panel flags it — compare
  against `Accessible` to see the fix.
- `Enter` and `Space` activate the button; `Tab` moves focus. The focus ring
  uses `:focus-visible` so it only shows for keyboard users.
- `loading` state sets `aria-busy` so screen readers announce the transition.
  Focus is preserved while loading.
